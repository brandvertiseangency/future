# Illustration and layout blocks (Behance-style guidance)

Use this when commissioning art or choosing stock for marketing and empty states.

## When to use custom illustration

- **2–4 signature moments only:** onboarding welcome, pricing hero, one empty state, optional error state.
- Prefer **one cohesive visual language** (line weight, color limited to brand blues + ink) across those moments.

## Block patterns that read “portfolio” vs “generic SaaS”

- **Full-bleed hero** with single strong image and large type; avoid three equal columns of icons.
- **60/40 split** — photography or UI crop on one side, copy on the other.
- **Pull quote** — short customer or brand line in `DM Serif Display` italic (`text-pull`).
- **Numbered case study** steps (01 / 02 / 03) instead of anonymous feature cards.

## Stock vs custom

- **Avoid:** repeated “team around laptop,” random isometric cubes, or mismatched illustration styles on every page.
- **OK:** abstract gradients using `#0188fd` → `#003bff` → `#001faa`, subtle noise, geometric shapes that match the new tokens.
- **Faces:** use real brand photography or none; do not mix multiple illustration face styles.

## Product screenshots

- Tight crops of **real UI** in light and dark theme; optional very subtle device frame or none.
- Prefer **one flagship screen** per section over many tiny thumbnails floating in gray voids.

## Iterate

- Ship **dashboard + landing** to final polish first, then clone the same blocks to inner pages.
